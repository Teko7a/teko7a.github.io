A. Add and Divide

题意

给定 \(a, b\)，你可对其进行如下两种操作：

  • a /= b (向下取整)
  • b += 1

问使得 \(a = 0\) 的最小操作次数。

解答

注意到第一种操作下降的非常快，因此一定是先执行第二种操作若干次，再执行第一种操作若干次 。

同时因为数据范围只有 \(2 ^ {30}\), 所以操作次数不会太大。故而枚举第二种操作进行了多少次，再进行模拟即可。

参考代码 
void solve() {
  int a, b;
  std::cin >> a >> b;

  int ans = 1E9;
  for (int i = 0; i < 30; ++i) {
    if (b + i != 1) {
      int res = i, x = a;
      while (x > 0) {
        x /= b + i;
        res += 1;
      }
      ans = std::min(ans, res);
    }
  }

  std::cout << ans << '\n';
}

复杂度： \(\mathcal O(\log^2 a)\)

B. Replace and Keep Sorted

题意

给定一个 递增 序列 \(\{a\}\)（元素 \(\in [1, k]\)，且互不相同 ），并同时定义长度相同的序列「\(k\)-相似」当且仅当仅有一个元素不同、递增、值域在 \(1 \sim k\)。

询问 \(q\) 次，\(a[l..r]\) 有多少个与之「\(k\)-相似」的序列（注意这个新序列无需在 \(\{a\}\) 中）。

解答

元素互异、递增、值域 \(1 \sim k\) 的限制使得此题的难度大减。

不妨从值域考虑，对于当前考虑的段 \(a[l..r]\)：

  1. 若要用 \(< a_l\) 或 \(> a_r\) 的元素替换掉端点满足题意，那么可以有 \((a_l - 1) + (k - a_r)\) 种方案。
  2. 而如果想要替换掉其中一个 \(a_i\)，则可以上调或者下调该元素（直到不与其他元素相同），这样的操作有 \(2 \times [a_r - a_l + 1 - (r - l + 1)]\)。

于是答案为

\(\begin{aligned} &a_l - 1 + k - a_r + 2a_r - 2a_l + 2 - 2r + 2l - 2 \\=\;& a_r - a_l + k - 2(r - l + 1) + 1\end{aligned}\)

展开代码 
for (int l, r, ans; q--; std::cout << ans << '\n') {
  std::cin >> l >> r;
  l -= 1, r -= 1;
  ans = a[r] - a[l] + k + 1 - 2 * (r - l + 1);
}

复杂度： \(\mathcal O(n + q)\)

C. Floor and Mod

题意

有多少二元组 \(\langle a, b \rangle\) 使得 \(\lfloor \dfrac{a}{b} \rfloor = a \bmod b\)，其中 \(a \in [1, x], b \in [1, y]\) ?

解答

一个经典结论是 \(\lfloor \dfrac{a}{b} \rfloor = a \bmod b \le \sqrt x\)。

\(\mathcal{Proof:\;\;} a \bmod b = a - b \times \lfloor \dfrac{a}{b} \rfloor \Rightarrow \dfrac{a}{b + 1} = \lfloor \dfrac{a}{b} \rfloor \Rightarrow a = k (b + 1)\)

\(k = a \bmod b \lt b \Rightarrow k^2 \lt k (b + 1) = a \le x \Rightarrow k \le \sqrt x \quad\quad \blacksquare\)

于是对于固定的 \(k\)，我们只需要计算满足下列条件的 \(b\) 的个数：

  1. \(b \gt k\)
  2. \(1 \le b \le y\)
  3. \(1 \le k(b + 1) \le x \Leftrightarrow 1 \le b \le x / k - 1\)

因此，答案为 \(\max(0, \min(y, x / k - 1) - k)\)。

展开代码 
for (int i = 1; i <= std::sqrt(x); ++i) {
  ans += std::max(0, std::min(y, (x - i) / i) - i);
}

复杂度： \(\mathcal O(\sqrt x)\)

D. Multiples and Power Differences

题意

关于矩阵 \(\{a\}\) 构造同样大的矩阵 \(\{b\}\)，使得：

  1. \(a_{i, j} \mid b_{i, j}\)
  2. \(b_{i, j} \le 10 ^ 6\)
  3. 任两相邻 \(\left| b_{i, j} - b_{x, y} \right| \le k ^ 4\)，其中 \(k \ge 1\)。
  • \(a_{i, j} \in [1, 16]\)。

解答

注意到 \(a_{i, j}\) 很小，但严苛的是 \(b_{i, j} \le 10 ^ 6\)，我们要找到一个数，恰好能包含 \(1 \sim 16\) 的所有数字。容易想到 \(\mathrm{lcm}\)，我们发现 \(\mathrm{lcm}{1..16} = 720,720\)。

于是就有一个简单的构造方案了：交叉分层放置，一层放 \(720,720\)，一层放 \(720,720 - a_{i, j}^4\) 即可。

展开代码 
auto X = 1LL;
for (int i = 1; i <= 16; ++i) {
  X = std::lcm(X, i);
}

int n, m;
std::cin >> n >> m;

for (int i = 0; i < n; ++i) {
  for (int j = 0, x; j < m; ++j) {
    std::cin >> x;
    if ((i + j) % 2 == 1) {
      std::cout << X - 1LL * x * x * x * x;
    } else {
      std::cout << X;
    }
    std::cout << " \n"[j + 1 == m];
  }
}

复杂度： \(\mathcal O(nm)\)

E. Move and Swap

题意

给定根节点为 \(1\) 的树，节点带有点权，且叶子到根的距离均为树高 \(d\)。

初始在根上放置两张卡片，一红一蓝（分别称为 \(a_r, a_b\)）。接下来，你将进行 \(d\) 次操作，每次操作分三个子步骤：

  1. 将红卡移动至其所在节点任一子节点。move(1/2)
  2. 将蓝卡移动至其所在节点下一层的任一节点。move(2/2)
  3. （可以跳过此步骤）交换两张卡片。swap

这样你可得到 \(\left|a_r - a_b \right|\) 分。

问最多能得到多少分？

解答

设 \(dp_i\) 为执行操作之后，当前位于节点 \(i\) 所收获的的最大值。由于第三步是可选的，所以先分开讨论：

  1. 不执行第三步：\(dp_i = dp_j + \max(\left| a_i - a_b\right|)\)，其中 \(a_b\) 为蓝卡所在节点的点权。用这一层的俩最值来转移一定最优，于是去掉绝对值就可以改写成 \(dp_i = dp_{p[i]} + \max(a_i - \min, \max - a_i)\)
  2. 执行第三步：假设换位之前红卡位于 \(j\)，则有： \(dp_i = \max(dp_i, dp_{p[j]} + \left| a_i - a_j \right|)\)。和上面类似地，\(dp_i = \max(dp_{p[j]} + a_j - a_i, dp_{p[j]} - a_j + a_i)\)。这种情况下我们需要得到 \(dp_{p[j]} \pm a_j\) 的最值。

这题的输入比较阴间，首先建出图之后用 dfs 处理出每个节点的父节点以及其深度，再把深度相同的节点存到一起。且注意到无论何种情况，都需要先知道 \(p_i\) 的情况，不妨直接从叶子开始 dp。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::vector<int> d(n);
  auto g = multivector<int>(n, 0, 0);

  for (int i = 1, u; i < n; ++i) {
    std::cin >> u;
    u -= 1;
    g[u].push_back(i);
    g[i].push_back(u);
  }

  for (int i = 1; i < n; ++i) {
    std::cin >> d[i];
  }

  std::vector<int> p(n, -1), dep(n);

  y_combinator([&](auto dfs, int u) -> void {
    for (auto &&v : g[u]) {
      if (v != p[u]) {
        dep[v] = dep[u] + 1;
        p[v] = u;
        dfs(v);
      }
    }
  })(0);

  int depth = *std::max_element(dep.begin(), dep.end());

  auto G = multivector<int>(depth + 1, 0, 0);

  for (int i = 0; i < n; ++i) {
    G[dep[i]].push_back(i);
  }

  std::vector<i64> dp(n);
  for (int i = depth; i >= 0; --i) {
#define inf 1E18
    i64 m = inf, M = -inf;
    int rm = 1E9, rM = -1E9;
    for (auto &&u : G[i]) {
      rm = std::min(rm, d[u]);
      rM = std::max(rM, d[u]);
      for (auto &&v : g[u]) {
        if (v != p[u]) {
          m = std::min(m, d[u] - dp[v]);
          M = std::max(M, d[u] + dp[v]);
        }
      }
    }

    m = std::min<i64>(m, rm);
    M = std::max<i64>(M, rM);

    for (auto &&u : G[i]) {
      dp[u] = std::max(d[u] - m, M - d[u]);
      for (auto &&v : g[u]) {
        if (v != p[u]) {
          auto ans = std::max(d[u] - rm, rM - d[u]);
          dp[u] = std::max(dp[u], dp[v] + ans);
        }
      }
    }
  }

  std::cout << dp.front() << '\n';
}

复杂度： \(\mathcal O(n)\)

F. Copy or Prefix Sum

题意

给定序列 \(\{B\}\)，关于之定义 「合法」 的序列 \(\{A\}\) 满足：

  1. Copy: \(B_i = A_i\)
  2. Prefix Sum \(B_i = \displaystyle\sum_{j=1}^i{\{A_j\}}\)

计算 \(\{A\}\) 的个数。

  • \(B_i \in [-10 ^ 9, 10 ^ 9]\)
  • \(n \in [1, 2 \times 10 ^ 5]\)

解答

用 \(dp_{\{i, j\}}\) 表示前缀长为 \(i\)、\(\{A\}\) 的前缀和为 \(j\) 的方案数，所求即 \(\displaystyle \sum _{j = -\infty}^{\infty} dp_{\{n, j\}}\)。

而对于转移：

  1. Copy: \(dp_{\{i, j\}} = dp_{\{i - 1, j - B_i\}}\)
  2. Prefix Sum: \(dp_{\{i, B_i\}} = \displaystyle\sum_{j = -\infty}^{\infty} dp_{\{i - 1, j\}}\)

（首先注意到第一维可以优化掉），第二种操作是简单的，只需要维护一下前缀即可快速转移，而对于第一种 全局修改 的操作却不能很快地维护。

这里就需要引入科技：Venice Technique（水位线技术）了！只需要维护一个全局懒标记 lv（也称水位线）就可以快速完成：

  1. 增 S.insert(lv + x)
  2. 删 S.erase(S.find(lv + x))
  3. 全局加 lv += x
  4. 得到最小值 *S.begin() - lv

一个带有懒标记的 multiset 而已。

如果使用 multiset 时间复杂度就是 \(\mathcal O(n \log n)\) 的了！若使用 unordered_map 还可进一步优化为 \(\mathcal O(n)\)。

展开代码 
static constexpr int mod = 1E9 + 7;

void solve() {
  int n;
  std::cin >> n;

  std::unordered_map<i64, i64, custom_hash> dp;

  dp[0] = 1;

  i64 s = 1, lv = 0, x;
  for (int i = 0; i < n; ++i) {
    std::cin >> x;
    i64 prefix = s - dp[lv];
    s = ((s + prefix) % mod + mod) % mod;
    dp[lv] = ((dp[lv] + prefix) % mod + mod) % mod;
    lv -= x;
  }

  std::cout << s << '\n';
}

复杂度： \(\mathcal O(n)\)
