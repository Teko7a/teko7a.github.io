A. Dense Array

题意

希望对于任两相邻元素都有：

\[ \dfrac{\max}{\min} \le 2 \]

在给定的数组中 插入 最少的数以满足上述条件。

\(n, a_i \le 50\)

解答

数据量非常小，直接模拟即可。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::vector<int> a(n);
  for (auto &i : a) {
    std::cin >> i;
  }

  int cnt = 0;
  for (int i = 0; i < n - 1; ++i) {
    int M = std::max(a[i], a[i + 1]);
    int m = std::min(a[i], a[i + 1]);
    while (M > 2 * m) {
      m *= 2;
      cnt += 1;
    }
  }
 
  std::cout << cnt << '\n';
}

复杂度： \(\mathcal O(n + \log n)\)

\(\log{\dfrac{a_2}{a_1}} +\log{\dfrac{a_3}{a_2}} + \log{\dfrac{a_4}{a_3}} + \cdots +\log{\dfrac{a_n}{a_{n - 1}}} = \log{\dfrac{a_n}{a_1}} \sim\log{n}\)

B. Balanced Remainders

题意

给定 \(n\) （其中 \(3 \mid n\)） 个数，记操作为选择一个数并使其自增 \(1\)，问使得模 \(3\) 意义下 \(cnt_0 = cnt_1 = cnt_2\) 的最小操作次数。

解答

易知最终的 \(cnt_x = \dfrac{n}{3}\)，加一将使得 \(0 \rightarrow 1 \rightarrow 2 \rightarrow 0\)，直接贪心地操作其中的最大值即可。

证明：只需说明这样操作不会比操作非最大值的代价更劣，而这一点显然。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::array<int, 3> cnt{};

  for (int i = 0, x; i < n; ++i) {
    std::cin >> x;
    cnt[x % 3] += 1;
  }

  int m = std::min({cnt[0], cnt[1], cnt[2]});

  for (auto &i : cnt) {
    i -= m;
  }

  int ans = 0;
  auto &[a, b, c] = cnt;

  while (!(a == b && b == c)) {
    ans += 1;
    auto M = std::max({a, b, c});
    if (a == M) {
      a -= 1, b += 1;
    } else if (b == M) {
      b -= 1, c += 1;
    } else {
      c -= 1, a += 1;
    }
  }

  std::cout << ans << '\n';
}

复杂度： \(\mathcal O(n)\)

注意到最坏情况下，此循环会执行 \(n\)次，构造 \(\langle\underbrace{2, 2,\cdots,2}_{length\; is\; n}\rangle\)。

使用势能分析的思想，每个元素的变化尺度可以从初末态直接得出，即 \(n\)。

C. Sum of Cubes

题意

是否有一种分解，使得给定的 \(n = a^3 + b^3\) ?

\(x \le 10^{12}\)

解答

此即 leetcode 第一题：两数之和，使用哈希表存下 \(\le 10 ^ 4\) 的立方数，随后枚举查询哈希表中 \(n - x^3\) 是否存在即可解决本题。

展开代码 
std::unordered_map<i64, bool, custom_hash> mp;

for (int i = 1; i <= 10000; ++i) {
  mp[1LL * i * i * i] = true;
}

int tt;
std::cin >> tt;
while (tt--) {
  [&]() {
    i64 n;
    std::cin >> n;
    for (int i = 1; i <= 10000; ++i) {
      i64 cur = 1LL * i * i * i;
      if (cur <= n) {
        if (mp.count(n - cur)) {
          return std::cout << "YES\n", void();
        }
      } else {
        break;
      }
    }
    std::cout << "NO\n";
  }();
}

复杂度： \(\mathcal O(n + {}^3\sqrt{10 ^ {12}})\)

D. Permutation Transformation

题意

某二叉树任意子树根节点权值比其他节点权值都大（堆）。

根据给定排列构造该树，设该数的根对应序列中 \(x\) 的位置。要求满足上述要求的情况下 \(x\) 的左子树包含 \(x\) 左区间中所有数，右子树包含右区间中的所有数（二叉搜索树）。

输出序列中每一个数对应节点的深度。

解答 1

一个朴素的想法是，直接 DFS，我们只需要找出区间最大值，将其作为根即可。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::vector<int> a(n), tree(n);
  for (auto &i : a) {
    std::cin >> i;
  }

  y_combinator([&](auto dfs, int l, int r, int d) -> void {
    if (l > r) {
      return;
    }
    int u = std::max_element(a.begin() + l, a.begin() + r + 1) - a.begin();
    tree[u] = d;
    dfs(l, u - 1, d + 1);
    dfs(u + 1, r, d + 1);
  })(0, n - 1, 0);

  for (auto &i : tree) {
    std::cout << i << ' ';
  }

  std::cout << '\n';
}

复杂度： \(\mathcal O(n ^ 2)\)

使用 ST / Segment tree等技巧寻找最值这一步，可以优化至 \(\mathcal O(n \log n)\)

解答 2

注意到此定义即「笛卡尔」 树（Cartesian tree）。

于是套用一下单调栈线性建树的模板，再 DFS 即可。

展开代码 
struct Node {
  int id, val, p, s[2];
  void init(int _id, int _val, int _p) {
    id = _id, val = _val, p = _p, s[0] = s[1] = 0;
  }
};

template<class Cmp>
int cartesian_build(std::vector<Node> &tree,
    const Cmp &&cmp = std::less<>()) {
  int n = tree.size() - 1;
  for (int i = 1; i <= n; ++i) {
    int k = i - 1;
    while (cmp(tree[i].val, tree[k].val)) {
      k = tree[k].p;
    }
    tree[i].s[0] = tree[k].s[1];
    tree[k].s[1] = i;
    tree[i].p = k;
    tree[tree[i].s[0]].p = i;
  }
  return tree[0].s[1];
}

void solve() {
  int n;
  std::cin >> n;

  std::vector<Node> tree(n + 1);
  tree[0].init(0, INT_MAX, 0);
  for (int i = 1, x; i <= n; ++i) {
    std::cin >> x;
    tree[i].init(i, x, 0);
  }

  int root = cartesian_build(tree, std::greater<>());
  std::vector<int> d(n + 1);

  y_combinator([&](auto dfs, int x) -> void {
    for (int i = 0; i < 2; ++i) {
      if (tree[x].s[i]) {
        d[tree[x].s[i]] = d[x] + 1;
        dfs(tree[x].s[i]);
      }
    }
  })(root);

  for (int i = 1; i <= n; ++i) {
    std::cout << d[i] << " \n"[i == n];
  }
}

复杂度： \(\mathcal O(n)\)

E. Accidental Victory

题意

\(n\) 人比赛，每位选手都有一些游戏币。在 \(n - 1\) 场 \(1V1\) 比赛中，拥有游戏币更多的选手取胜，并获得输家的全部游戏币。特殊的，如果两种拥有相同的游戏币，那么取胜看天（随机决出胜者）。

问可能有多少选手成为最终的赢家？

注意，原题面中并非「游戏币」，而是tokens。真不知道咋翻译比较合适，于是就取「代币」 之意了。

解答

考虑 \(x\) 将会成为最终的赢家，并先逐个击破比他小的选手，随后向右拓展，只要能够赢下最后一个人就能够成为赢家。特殊的，如果当前拥有的游戏币与对手相同，我们都视作 \(x\) 会赢（这样的假设是必要的）。

注意到如果 \(x\) 能成为赢家，那么使用此策略 \(x + 1\) 也能成为赢家（要保证这一点始终成立，需要提前排序）。此即二分答案基本问题。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::vector<std::pair<i64, int>> a(n);
  for (int i = 0; i < n; ++i) {
    std::cin >> a[i].first;
    a[i].second = i + 1;
  }

  std::sort(a.begin(), a.end());

  int l = 0, r = n;
  while (l < r) {
    int m = (l + r) / 2;

    auto check = [&]() {
      i64 cur = 0;
      for (int i = 0; i < m; ++i) {
        cur += a[i].first;
      }
      for (int i = m; i < n; ++i) {
        if (cur < a[i].first) {
          return false;
        }
        cur += a[i].first;
      }
      return true;
    }();

    if (check) {
      r = m;
    } else {
      l = m + 1;
    }
  }

  std::vector<int> ans;
  for (int i = l - 1; i < n; ++i) {
    ans.push_back(a[i].second);
  }

  std::cout << ans.size() << '\n';

  std::sort(ans.begin(), ans.end());
  for (auto &i : ans) {
    std::cout << i << ' ';
  }

  std::cout << '\n';
}

复杂度： \(\mathcal O(n \log n)\)

Bonus: 其实要获得答案无需二分，从后往前遍历，比较前缀和与 \(x\) 的相对大小也可直接得出哪些可行。

复杂度上没有太大的影响。

F. Equalize the Array

题意

在给定的数组中删去至少几个元素，能使得剩下的元素都出现 \(x\) 次？

解答

不妨考虑最终最多剩下几个元素，答案就是 \(n -\) 剩下的。

统计每个元素出现次数，并对次数降序排序，随后求解最大子矩形。需要指出的是，在有序数组上做最大子矩形是容易的。

展开代码 
void solve() {
  int n;
  std::cin >> n;

  std::unordered_map<int, int, custom_hash> mp;
  for (int i = 0, x; i < n; ++i) {
    std::cin >> x;
    mp[x] += 1;
  }

  std::vector<int> a;
  for (auto &[k, v] : mp) {
    a.push_back(v);
  }

  std::sort(a.rbegin(), a.rend());

  int ans = 0;
  for (int i = 0; i < (int)a.size(); ++i) {
    ans = std::max(ans, (i + 1) * a[i]);
  }

  std::cout << n - ans << '\n';
}

复杂度： \(\mathcal O(n \log n)\)

G. Old Floppy Drive

题意

给定一个循环数组（周期为 \(n\)），接下来进行 \(q\) 次询问首个前缀和 \(\ge x\) 的位置。

如果不存在这样的位置，输出 \(-1\)。

\(a_i \in [-10 ^ 9, 10 ^9]\)，注意开 long long.

解答

很明显需要二分查找。但注意到有负数，要进行二分，需要保证有序，这该怎么办呢？

我们可以选择只维护那些前缀和递增的点，同时记下整个数组的和 \(s\)。

\(p_{\{-1, 0\}}\) 是 \(p\) 数组末尾元素的第一维的值。

如果 \(s \le 0\)，且询问的 \(x \gt p_{\{-1, 0\}}\) 是无解的，因为在循环几圈之后，前缀和只会变得更小。

否则 \(x \le p_{\{-1, 0\}}\)，可以直接 lower_bound 查找这个值。

另外的情况就是 \(s \gt 0\)，这种时候需要先减去已经跑了多少圈，即：

\[ \Bigg\lceil \dfrac{x - b}{s} \Bigg\rceil \]

再 lower_bound \(x - r \times s\) 得到位置 \(pos\)，再加上 \(r \times n\) 就是最终的答案了。

展开代码 
void solve() {
  int n, q;
  std::cin >> n >> q;

  i64 s = 0;
  std::vector<std::pair<i64, int>> p;

  for (int i = 0, x; i < n; ++i) {
    std::cin >> x;
    s += x;
    if (!i || s > p.back().first) {
      p.emplace_back(s, i);
    }
  }

  for (i64 x; q --; std::cout << " \n"[!q]) {
    std::cin >> x;

    auto &b = p.back().first;
    if (s <= 0) {
      if (x > b) {
        std::cout << "-1";
      } else {
        std::cout << std::lower_bound(p.begin(), p.end(), std::pair{x, 0})->second;
      }
    } else {
#define CEIL(a, b) (a + b - 1) / b
      i64 r = std::max(0LL, CEIL(x - b, s));
      std::cout << std::lower_bound(p.begin(), p.end(), std::pair{x - r * s, 0})->second + r * n;
    }
  }
}

复杂度： \(\mathcal O(n + q \log n)\)
